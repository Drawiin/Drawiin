# Modularization Structure - Neon

## Context
Implementation of a consistent modularization structure to promote code sharing between native and hybrid apps.

## Challenge/Problem
There was a need to optimize development and increase code reuse between native and hybrid applications. The existing structure didn't promote efficient code sharing.

## Your Role
Android Developer - Contributing to the development of essential features and improvements for Neon's Android banking apps, impacting more than 10 million clients.

## Actions Taken
- Implemented a consistent modularization structure
- Promoted code sharing between native and hybrid apps
- Optimized development processes
- Increased code reuse

## Results
- **Optimized development** through better code organization
- **Increased code reuse** between native and hybrid apps
- Better maintainability
- Improved scalability

## Technologies Used
- Android Modularization
- Kotlin
- Gradle
- Multi-module architecture
