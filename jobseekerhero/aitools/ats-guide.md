# ATS Optimization Guide

Guide to optimizing your CV for Applicant Tracking Systems (ATS).

## ü§ñ What is ATS?

Applicant Tracking Systems are software used by companies to manage job applications. They:
- Parse and store resumes
- Filter candidates based on keywords
- Rank applicants by match score
- Help recruiters manage the hiring process

**Why it matters:** If your CV isn't ATS-friendly, it might never reach a human recruiter.

## ‚úÖ ATS-Friendly Formatting

### Do's
- ‚úÖ **Use standard fonts** - Arial, Calibri, Times New Roman, Georgia
- ‚úÖ **Use standard headings** - "Work Experience", "Education", "Skills"
- ‚úÖ **Save as PDF or Word** - Both work, but PDF is preferred
- ‚úÖ **Use simple formatting** - Bullet points, bold, italics (sparingly)
- ‚úÖ **Include keywords** - From the job description
- ‚úÖ **Use standard date formats** - "MM/YYYY" or "Month YYYY"
- ‚úÖ **Keep it simple** - Clean, straightforward layout

### Don'ts
- ‚ùå **Avoid graphics/images** - ATS can't read them
- ‚ùå **No tables or columns** - ATS struggles with complex layouts
- ‚ùå **Avoid headers/footers** - Can be cut off by ATS
- ‚ùå **No fancy fonts** - Stick to standard fonts
- ‚ùå **Avoid text boxes** - ATS can't parse them well
- ‚ùå **No special characters** - Use standard punctuation

## üîë Keyword Optimization

### How to Find Keywords
1. **Read the job description** carefully
2. **Identify required skills** - Technical and soft skills
3. **Note industry terms** - Domain-specific language
4. **Look for repeated phrases** - These are likely important
5. **Check similar job postings** - Find common keywords

### Where to Place Keywords
- **Skills section** - List relevant skills
- **Work experience** - Use keywords naturally in descriptions
- **Summary/Objective** - Include key terms
- **Job titles** - Match job title terminology when possible

### Keyword Strategy
- **Use exact phrases** from job description when natural
- **Include variations** - "Android" and "Android Development"
- **Don't keyword stuff** - Use keywords naturally
- **Match their language** - Use their terminology

## üìù ATS-Friendly CV Structure

### Recommended Sections (in order)
1. **Contact Information** - Name, phone, email, LinkedIn
2. **Professional Summary** - 2-3 sentences with keywords
3. **Skills** - List of relevant skills
4. **Work Experience** - Reverse chronological
5. **Education** - Degrees and certifications
6. **Additional Sections** - Projects, certifications, languages (if relevant)

### Section Headings
Use standard headings that ATS recognizes:
- "Work Experience" or "Professional Experience"
- "Education"
- "Skills" or "Technical Skills"
- "Certifications"
- "Projects" (if applicable)

## üß™ Testing Your CV

### ATS Testing Tools
- **Jobscan** - https://www.jobscan.co
- **Resume Worded** - https://resumeworded.com
- **VMock** - https://www.vmock.com

### Manual Testing
1. **Copy text from your CV** - Paste into a text editor
2. **Check formatting** - Is it readable as plain text?
3. **Search for keywords** - Can you find important terms easily?
4. **Test with job description** - Compare your CV to job requirements

## üí° Pro Tips

1. **Tailor for each application** - Match keywords to each job
2. **Use both full terms and acronyms** - "Android" and "Android SDK"
3. **Include location** - If job posting mentions location
4. **Match job title** - Use similar terminology to job posting
5. **Keep it scannable** - Both for ATS and humans
6. **Update regularly** - Add new skills and keywords as you learn them

## üö´ Common ATS Mistakes

- Using only images/charts for important information
- Complex layouts with multiple columns
- Unusual file formats (use PDF or Word)
- Missing contact information
- Not including keywords from job description
- Using non-standard section headings
- Overly creative formatting that confuses ATS

## ‚úÖ Checklist

Before submitting your CV:
- [ ] Saved as PDF or Word format
- [ ] Uses standard fonts (Arial, Calibri, Times New Roman)
- [ ] No images, graphics, or charts

- [ ] Contact information clearly visible
- [ ] Tested with ATS checker tool

*Remember: Your CV needs to pass both ATS and human review. Optimize for ATS, but keep it readable for humans too.*
